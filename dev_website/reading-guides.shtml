Organization

Our guides each cover a distinct topic. Some are more of a "how-to", with a list of specific instructions; others include suggestions but not explicit instructions, and others simply contain general or reference information.

Pre-formatted text

There are different kinds of pre-formatted text in our guides.

  • This text box: 
    universe = docker
    Indicates information that should be placed into a submit file
  • This text box: 
    #!/bin/bash
    Indicates any other file -- a text file, user code, or a script.
  • This text box: 
    [alice@submit]$ condor_q
    Shows commands that should be entered in the command line or terminal interface that appears when you have logged into a submit server or head node. Note that the first part of the text, up to the $ is what is called the "prompt". This isn't something that you type, but standard text that appears at the start of each line waiting for a new command.
We use blocks like this to indicate instructions that may be relevant to some people, but not all, or to offer general advice.

Customization

Certain commands or files in our guides can (and should!) be copied verbatim. However, most commands or files will need some changes, using information like your username, your file or script name, or your job's ID. This information is noted in italicized text, like so: 

[alice@submit]$ condor_q JobId
